# レジ締め結果閲覧ページ 仕様書

## 1. 概要

レジ締め完了後の精算結果を、公演単位で一覧・詳細確認できるページ。
各公演回の最新レジ締め結果をサマリーで表示し、詳細や過去の履歴も確認可能にする。

---

## 2. ページ構成

### URL: `/productions/[id]/cashclose-report`

公演（Production）単位のページとし、全公演回のレジ締め状況を一覧表示する。
個別の公演回の詳細はページ内でアコーディオン展開またはカード展開で表示する（ページ遷移なし）。

---

## 3. アクセス導線

| 場所 | 方法 |
|------|------|
| **ダッシュボード** | メニューグリッドに「レジ締めレポート」カードを追加（レポートカードの隣） |
| **レジ締め完了画面** | 「レジ締め完了」表示に「レジ締めレポートを見る」リンクを追加 |
| **売上レポートページ** | ページ下部に「レジ締めレポート」へのリンクを追加 |
| **チェックインページ** | 既存の「レジ締め」ボタンの横に「レジ締め結果」リンクを追加 |

## 5. データ取得

### 必要なデータ

1. **公演情報**: `productions/{id}` - タイトル、ticketTypes
2. **全公演回**: `performances` where `productionId == id` - 開演日時、ID
3. **全レジ締め**: `cashClosings` where `productionId == id` && `userId == uid` - 公演回ごとにグループ化

### クエリ

```typescript
// 全公演回のレジ締めデータを一括取得（productionId単位）
const q = query(
    collection(db, 'cashClosings'),
    where('userId', '==', userId),
    where('productionId', '==', productionId),
    orderBy('createdAt', 'desc')
);
```

取得後にクライアント側で `performanceId` ごとにグループ化し、各グループの先頭（最新）を表示用に使う。

### 必要な複合インデックス

既存の `cashClosings` インデックス（userId + performanceId + productionId + createdAt）でカバー可能。
ただし、performanceId を含まないクエリ（productionId 単位）を使う場合は別途インデックスが必要:

```
cashClosings: userId ASC, productionId ASC, createdAt DESC
```

## 7. UI デザイン方針

- 既存の `SalesReportView` のレイアウトパターンに合わせる
- サマリーカード: `display: grid, gridTemplateColumns: repeat(auto-fit, minmax(200px, 1fr))`
- 公演回カード: `.card` + 展開/折りたたみ（useState で管理）
- maxWidth: `1000px`（売上レポートと同じ）
- 差額の色分け: 一致=`var(--success)` / 多い=`#1565c0` / 不足=`var(--accent)`
- ステータスバッジ: 精算済み=緑背景 / 未精算=グレー背景

---

## 8. 権限

- 主催者（Google ログイン、production.userId === user.uid）のみアクセス可能
- スタッフはこのページにはアクセスしない（レジ締め実行のみ）
