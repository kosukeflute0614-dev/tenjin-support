# UI/UX 改善チェックリスト

**対象**: Tenjin-Support（劇場制作支援アプリ）
**作成日**: 2026-03-12

このチェックリストは、アプリの画面上で目に見える部分の改善点をまとめたものです。
優先度順に並べています。

---

## 優先度：高（使いやすさに直接影響）

### 読み込み中の表示が文字だけ

- [ ] **全般**: データを読み込んでいる間、「読み込み中...」という文字しか表示されない。回転するアイコンやアニメーションがないため、アプリが動いているのかフリーズしているのか判断しづらい
  - 該当する画面：ダッシュボード、予約一覧、公演一覧、スタッフポータル、オンボーディング
  - **改善案**: 文字だけでなく、回転するスピナーやプログレスバーを表示して「動作中」であることを視覚的に伝える

### ダークモード時の見づらさ（複数箇所）

- [ ] **FAQページ**: ダークモード（暗い画面テーマ）にすると、質問欄の背景が白のままになっている。暗い画面に白い四角が浮いて見えて違和感がある
  - **改善案**: 背景色をダークモード用の暗い色に切り替わるようにする

- [ ] **一覧表のヘッダー行**: ダークモード時、テーブル（表）の見出し行の色が本体と区別しにくくなる。見出しなのかデータなのか一瞬わからない
  - **改善案**: ダークモード時でも見出し行とデータ行の色がはっきり区別できるようにする

- [ ] **お問い合わせページ**: ダークモード時、入力欄の枠線がほぼ見えなくなる。どこに文字を入力すればいいのか分かりにくい
  - **改善案**: 入力欄の枠線の色をダークモードに対応させる

- [ ] **ランディングページのヘッダー**: 画面をスクロールした時にヘッダー（画面上部のナビゲーション）の背景が白固定になっている。ダークモード時に白いバーが画面上部に残り浮いて見える
  - **改善案**: ダークモード時にはヘッダー背景も暗い色にする

- [ ] **「データがありません」の表示**: 一覧画面でデータがない時に表示される点線の枠が、ダークモード時にほぼ見えなくなる
  - **改善案**: 枠線の色をダークモードに対応させる

---

## 優先度：中（体験の一貫性に影響）

### 日本語化されていない部分

- [ ] **公開予約フォームのヘッダー**: 予約ページの上部に「TICKET RESERVATION」と英語で表示されている。他の部分は全て日本語なので浮いている
  - **改善案**: 「チケット予約」など日本語に変更する

### エラー画面が用意されていない

- [ ] **存在しないページにアクセスした時**: アプリのデザインとは全く異なるシンプルな404ページ（ページが見つかりません）が表示される。ユーザーがアプリから離れたように感じる
  - **改善案**: アプリのデザインに合わせた「ページが見つかりません」画面を用意する

- [ ] **予期しないエラーが発生した時**: 画面全体が真っ白になりリカバリできない。進行中の作業が失われる
  - **改善案**: 「エラーが発生しました。再読み込みしてください」のようなメッセージとボタンを表示する画面を用意する

### メール送信失敗がわからない

- [ ] **予約完了後のメール**: 予約自体は成功しても確認メールの送信に失敗した場合、ユーザーには何も通知されない。メールが届かないのに気づけない
  - **改善案**: メール送信に失敗した場合は「予約は完了しましたが、確認メールの送信に失敗しました」などのメッセージを表示する

---

## 優先度：低（仕上げ・細かな改善）

### スタッフログインのセキュリティ

- [ ] **パスコード入力**: スタッフログイン画面で、パスコードを何度間違えても制限なく再入力できる。悪意のある人が手当たり次第に試せてしまう
  - **改善案**: 5回間違えたら「30分後に再試行してください」のようなロックをかける

### 初期状態の案内

- [ ] **データが空の時の案内**: ダッシュボードや各一覧画面に初めてアクセスした時、ただ空欄が表示されるだけの箇所がある。次に何をすればいいかわからない
  - **改善案**: 「まだ公演が登録されていません。新しい公演を作成しましょう」のような案内メッセージとボタンを表示する

### 入力フォームの細かな改善

- [ ] **オンボーディング（初期設定）の日時入力**: 公演日時の入力欄に、入力すべき形式（例：2026/04/01）のヒントが表示されない。正しい形式がわからず入力に迷う可能性がある
  - **改善案**: 入力欄内に薄い文字で入力例を表示する（プレースホルダ）

- [ ] **任意・必須の表示**: フォーム画面で「必須」「任意」の表示スタイルが画面によって微妙に異なる
  - **改善案**: 全フォームで統一したスタイルにする

### モバイル画面の細かな問題

- [ ] **ランディングページのメニュー**: スマートフォンで画面の横幅が非常に狭い場合（320px以下の端末）、メニューパネルが画面からはみ出す可能性がある
  - **改善案**: メニュー幅を画面幅に応じて自動調整する

---

## 良好な点（変更不要）

以下の点は現状で問題なく実装されています：

- スマホ・タブレット・パソコンの3段階で画面レイアウトが切り替わる（レスポンシブ対応）
- キーボード操作やスクリーンリーダーへの配慮がある（アクセシビリティ）
- 日本語化はほぼ完全（上記の一部を除く）
- 公開予約フォームの必須入力表示が適切
- スタッフログインのパスコード入力欄が大きく見やすい
- リアルタイムデータ更新時のメモリリークがない（技術的に適切）
- 読み込み完了後の表示は全般的にスムーズ
